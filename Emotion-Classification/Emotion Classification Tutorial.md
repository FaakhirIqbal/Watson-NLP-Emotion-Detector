<p>With IBM Watson NLP, IBM introduced a common library for artificial intelligence (AI) runtimes (for serving the model) and AI libraries (like Natural Language Processing, Document Understanding, Translation, and Trust). IBM Watson NLP brings everything under one umbrella for consistency and ease of development and deployment. This tutorial walks you through the steps of using a pretrained model to classify emotions in tweets using the <code>watson_nlp</code> library from IBM Watson NLP.</p>
<p>The <code>watson_nlp</code> library is available on IBM Watson Studio as a runtime library so that you can directly use it for model training, evaluation, and prediction. The following figure shows the Watson NLP NLP architecture.</p>
<p><a target="_blank" rel="noopener noreferrer" href="/Shivam-Raj-Solanki/Code-Tutorials/blob/9f4785afe8a6506464eadf8625daa4cc0d933b94/use-the-watson-nlp-library-to-perform-emotion-classification/images/Watson-Studio-flow.png"><img src="/Shivam-Raj-Solanki/Code-Tutorials/raw/9f4785afe8a6506464eadf8625daa4cc0d933b94/use-the-watson-nlp-library-to-perform-emotion-classification/images/Watson-Studio-flow.png" alt="Architecture" style="max-width:100%;"></a></p>
<p>IBM Watson NLP is a standard embeddable AI library that is designed to tie together the pieces of IBM Natural Language Processing. It provides a standard base natural language processing layer along with a single integrated roadmap, a common architecture, and a common code stack designed for widespread adoption across IBM products.</p>
<p>Emotion Classification is usually manually processed by humans to gather groups of qualitative data. Having the ability to automatically gather and process larger data sets of text through customer feedback, comments, or an entire article written on your product is a strong tool to gain insight into the most common emotional responses in a group of people or a block of text.</p>
<p>IBM Watson NLP now provides the ability to automatically classify text into the strongest emotions that are typically tracked: sadness, joy, anger, fear, and disgust. We are able to infer upon a certain emotion through syntax analysis and the emotion workflows provided by IBM AI libraries by using pretrained and custom models. The following figure shows emotion classification predictions.</p>
<p><a target="_blank" rel="noopener noreferrer" href="/Shivam-Raj-Solanki/Code-Tutorials/blob/9f4785afe8a6506464eadf8625daa4cc0d933b94/use-the-watson-nlp-library-to-perform-emotion-classification/images/ensemble-model-prediction.png"><img src="/Shivam-Raj-Solanki/Code-Tutorials/raw/9f4785afe8a6506464eadf8625daa4cc0d933b94/use-the-watson-nlp-library-to-perform-emotion-classification/images/ensemble-model-prediction.png" alt="Ensemble model prediction" style="max-width:100%;"></a></p>

<pre><code>def extract_emotion(text):
    # run the emotion model on the result of the syntax analysis
    emotion_result = emotion_model.run(text, document_emotion=True)
    
    document_emotion = emotion_result.to_dict()['emotion_predictions'][0]['emotion']
    mention_emotion = [(sm['span']['text'], sm['emotion']) for sm in emotion_result.to_dict()['emotion_predictions'][0]['emotion_mentions']]
    return (document_emotion, mention_emotion)

# Helper method to create a new dataframe with the corresponding emotion
def create_emotion_dataframe(df):
    emotion = df['text'].apply(lambda text: extract_emotion(text))
    emotion_df = pd.DataFrame.from_records(emotion, columns=('Document emotion', 'Mention emotion'))
    return emotion_df
